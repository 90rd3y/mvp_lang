### `GRAMMAR.md`:

#### Лексика

1) Нужен полный список всех ключевых слов вместо ленивого с "и т.д."
2) Типы определены как идентификаторы, а не ключевые слова?
3) Не указаны такие арифметические операции, как умножение и деление.
4) Не указано логическое отрицание. И есть ли оно в языке?
5) Не указаны квадратные скобки.
6) Не указана точка (обращение к полям структуры)
7) **Определены ли пространства имен и оператор его разрешения? Должно быть определено ,ибо это обязательное требование ТЗ**

#### Синтаксис

1) Грамматика основана на рекурсивном спуске ДЛЯ ОПЕРАТОРОВ? Возможно для объявлений и инструкций, а не для операторов требуется уточнение.
2) "и на Pratt парсере для математических выражений?" Может быть для выражений в целом? Уточнить.
3) `stmt` должен быть вынесен на уровень выше, он не должен являться инстанцией `Decl`.
4) Должен поддерживаться прототип функции, чтобы удовлетворить требование ТЗ, чтобы в языке использовались объекты (переменные, функции) только после объявления. Иначе это статическая ошибка.
5) `Vardecl` допускает многомерные массивы. Учитывая, что в языке нет указателей (их не должно быть для простоты реализации), логичноепредположение, что нельзя допускать многомерные массивы. Нужно оставить только одномерные массивы с простыми базовыми типами.
6) `vardecl` допускает непроинициализированные переменные. Это должно быть запрещено для простоты и надежности.
7) `Block` является блоком инструкций. На этом уровне должы быть запрещены все объявления, кроме объявления переменной. Блок должен содержать инструкции, а не объявления, вообще говоря.
8) Настоятельно рекомендуется запретить инструкции ,являющиеся выраженимем. Определить надо только жестко отдельный специальную инструкцию вызова функции. Требуется для надежности.
9) Список встроенных функций оформить отдельным документом, ибо вообще говоря не является непосредственной частью грамматики. Более того, наилучшим будет, если вызов всех встроенных функций будет определен одним грамматическим правилом (в данном случае что-то типа `Indentifier` "(" `Expr` ("," `Expr`)? ")")
10) Присваивание должно являться исключительно инструкцией, оно недопустимо как инстанция выражения. Нужно для надежности.
11) Литералы можно объединить под общей граматической единицей ,а не всякий раз обращаться к каждому в остальных частях грамматики.
