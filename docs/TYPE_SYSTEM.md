# Система Типов

Этот документ описывает систему типов, правила проверки и приведения типов в учебном языке.

## 1. Строгая Статическая Типизация

Язык использует **строгую статическую типизацию** (Strict Static Typing).
1. Тип каждой переменной, выражения и функции известен на этапе компиляции (в фазе семантического анализа).
2. Запрещены любые неявные приведения типов (Implicit Conversions / Coercions). Нельзя неявно сложить `целое` и `вещ`, или использовать `целое` в `если (условие)`.
3. Каждая математическая и логическая операция проверяет совместимость типов перед тем как пропустить код в этап интерпретации.

## 2. Базовые типы (Примитивы)

| Ключевое слово | Аналог в C++ | Описание |
|----------------|--------------|----------|
| `целое`        | `int64_t`    | 64-битное знаковое целое число. |
| `вещ`          | `double`     | 64-битное число с плавающей точкой. |
| `лог`          | `bool`       | Логическое значение (`истина` или `ложь`). |
| `строка`       | `char*`      | UTF-8 нуль-терминированная строка. |
| `символ`       | `char`       | Одиночный байтовый/UTF-8 символ. |
| `пусто`        | `void`       | Отсутствие типа. Используется как возвращаемое значение процедур. |

## 3. Пользовательские типы

Система типов расширяема и позволяет пользователю строить собственные абстракции.

### 3.1 Синонимы типов (Type Aliases)
```text
тип Число = целое;
тип Матрица = Число; // Синоним на синоним
```
Синонимы типов не создают новый тип (в отличие от `newtype` в некоторых языках), а являются "прозрачными". Везде, где ожидается `целое`, можно передать `Число`.
Семантический анализатор реализует алгоритм **Unwrapping** (раскрутки) алиасов: при проверке типов он рекурсивно доходит до базового типа (основания) и сравнивает именно их.

### 3.2 Массивы (Static Arrays)
```text
целое[10] массив;
```
Тип "массив" содержит две составляющие: базовый тип элементов и статический размер, зафиксированный в коде. В текущем MVP многомерные массивы напрямую не поддерживаются в синтаксисе, но могут быть эмулированы. Массивы аллоцируются в Арене и инициализируются значениями по умолчанию.
Индексация `массив[индекс]` требует, чтобы `индекс` строго имел тип `целое`.

### 3.3 Структуры (Structs)
```text
структура Точка { целое х; целое у; }
```
Структуры — это пользовательские составные типы данных. Каждое объявление структуры регистрирует в глобальной `TypeTable` уникальный TypeID. 
В отличие от синонимов, две структуры с одинаковыми наборами полей — это **два разных типа** (Nominal Subtyping, а не Structural Subtyping). 

Поддерживается неограниченная вложенность структур:
```text
структура Игрок { Точка позиция; }
```
При доступе `Игрок.позиция.х` система типов проверяет цепочку типов, гарантируя, что каждое поле существует и возвращает свой строгий тип.

## 4. Правила приведения и проверок в выражениях

* **Присваивание (`=`)**: Тип левой части (L-Value) обязан строго совпадать с типом правой части (R-Value). 
* **Арифметика (`+`, `-`, `*`, `/`, `%`)**: Операнды должны иметь одинаковый тип (либо два `целое`, либо два `вещ`). Возвращает тот же тип. Сложение строки и числа приведет к ошибке компиляции.
* **Сравнения (`==`, `!=`, `>`, `<`, `>=`, `<=`)**: Операнды должны иметь одинаковый тип. Возвращают тип `лог`.
* **Логика (`&&`, `||`, `!`)**: Операнды обязаны иметь тип `лог`. Возвращает `лог`.

## 5. Внутреннее устройство (`TypeTable`)

Во время работы семантического анализатора все встреченные типы регистрируются в `TypeTable`. 
Каждый тип имеет `TypeId` (числовой идентификатор). 
Внутреннее представление типа:
```cpp
struct Type {
    TypeKind kind;        // Примитив, Структура, Массив, Алиас
    Lexer::IdentId name;  // Имя (интернированная строка)
    uint32_t size;        // Размер массива (для TypeKind::Array)
    TypeId base_type;     // Базовый тип (для Алиасов и Массивов)
};
```
Использование глобальной таблицы типов и числовых идентификаторов избавляет компилятор от необходимости глубокого сравнения строк и структур при тайп-чекинге, делая проверку типов `O(1)`.
