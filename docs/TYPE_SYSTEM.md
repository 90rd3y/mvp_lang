# Система Типов

Этот документ описывает систему типов, правила проверки и приведения типов в учебном языке на основе реальной реализации семантического анализатора (`Semantic.cpp`).

## 1. Строгая Статическая Типизация

Язык использует **строгую статическую типизацию** (Strict Static Typing).
1. Тип каждой переменной, выражения и функции разрешается и проверяется на этапе статического анализа (до выполнения программы).
2. Запрещены любые неявные приведения типов (Implicit Conversions / Coercions).
3. Присваивание, вызовы функций и бинарные операции требуют строгого соответствия типов (реализовано через функцию `types_compatible`).

## 2. Базовые типы (Примитивы)

| Ключевое слово | Назначение |
|----------------|------------|
| `целое`        | Целочисленный знаковый тип. |
| `бцелое`       | Целочисленный беззнаковый тип. |
| `вещ`          | Число с плавающей точкой. |
| `лог`          | Логическое значение (`истина` или `ложь`). |
| `строка`       | Строковый тип. |
| `символ`       | Одиночный символ. |
| `пусто`        | Отсутствие типа (используется как возвращаемое значение функций, не возвращающих данных). |

## 3. Пользовательские типы

### 3.1 Синонимы типов (Type Aliases)
```text
тип Число = целое;
тип Матрица = Число; // Синоним на синоним
```
Синонимы типов являются "прозрачными". При любой проверке совместимости семантический анализатор рекурсивно "раскручивает" (unwraps) синонимы вплоть до базового типа (через внутреннюю функцию `resolve_alias`). Таким образом, тип `Число` полностью взаимозаменяем с `целое`.

### 3.2 Массивы (Static Arrays)
```text
целое[10] массив = [1, 2, 3];
```
Массивы определяются базовым типом и фиксированным размером. 
* Два массива считаются совместимыми, только если совпадают их базовые типы (с учетом алиасов) **и их размеры**.
* Индексация (`arr[i]`) строго требует, чтобы тип выражения индекса был `целое`.
* При инициализации массива через литерал (`[1, 2, 3]`), пустые массивы (`[]`) запрещены. Тип всех последующих элементов в литерале обязан строго совпадать с типом самого первого элемента.
* В текущем MVP многомерные массивы напрямую не поддерживаются в синтаксисе, но могут быть эмулированы. Массивы аллоцируются в Арене и инициализируются значениями по умолчанию.

### 3.3 Структуры (Structs)
```text
структура Точка { целое х; целое у; }
```
Каждая структура регистрируется в таблице типов как уникальный идентификатор (Nominal Typing). 
Две структуры с одинаковыми именами полей и их типами считаются **несовместимыми**, если это разные структуры.
Поддерживается вложенность структур. Рекурсивное объявление структуры внутри самой себя запрещено и вызывает ошибку.

**Внимание (Особенность реализации):**
При инициализации структуры через литерал (`Точка { 1, 2 }`) семантический анализатор в настоящее время выполняет только **упрощенную проверку**: он проверяет лишь то, что количество переданных в литерал аргументов совпадает с количеством полей структуры. *Проверка совпадения типов отдельных полей в литерале в текущей версии анализатора не реализована.*

## 4. Явное приведение типов (CastExpr)
Синтаксис приведения: `как <Тип>(Выражение)`.
На уровне семантического анализатора операция приведения типа выполняет принудительную подмену возвращаемого типа выражения без каких-либо проверок совместимости.

## 5. Правила проверок в выражениях

* **Присваивание (`=`)**: Тип левой части (L-Value) обязан быть строго совместимым с типом правой части (R-Value) после раскрутки алиасов.
* **Бинарные операторы**:
  * Операнды всегда должны иметь одинаковый тип (с учетом алиасов). Нельзя, например, сложить `целое` и `вещ`.
  * **Операторы отношения** (`<`, `>`, `<=`, `>=`): Допускается их применение **только** к типам `целое`, `вещ` и `символ`. Сравнение беззнаковых чисел (`бцелое`), строк, логических значений, массивов и структур вызывает статическую ошибку. Возвращают тип `лог`.
  * **Операторы равенства** (`==`, `!=`): Применимы ко всем типам при условии строгого совпадения типов левого и правого операндов. Возвращают `лог`.
  * **Арифметические операторы** (`+`, `-`, `*`, `/`, `%`): Разрешены только для совпадающих типов. Возвращают тип операндов.

## 6. Внутреннее устройство (`TypeTable`)

Во время работы семантического анализатора все встреченные типы регистрируются в глобальной таблице `TypeTable`. Каждый тип имеет целочисленный идентификатор `TypeId`.
Использование глобальной таблицы типов и числовых идентификаторов избавляет интерпретатор от необходимости глубокого сравнения строк и структур при тайп-чекинге, делая проверку типов `O(1)`.
