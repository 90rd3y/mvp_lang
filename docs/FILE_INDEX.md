# Карта проекта (Project Index)

Этот файл описывает структуру директорий и предназначение файлов в проекте. **Агент обязан придерживаться этой структуры при дальнейшей работе.**

## Исходный код (C++)
* `inc/` — Заголовочные файлы (`.hpp`).
  * `Memory.hpp` — Арена-аллокатор.
  * `Lexer.hpp` — Лексический анализатор и пул строк.
  * `Parser.hpp` — Синтаксический анализатор (Flat AST, Pratt Parser).
  * `Semantic.hpp` — Семантический анализатор (проверка типов, области видимости).
  * `Interpreter.hpp` — Виртуальная машина (Tree-walking, стек вызовов).
* `src/` — Файлы реализации (`.cpp`).
  * `main.cpp` — Точка входа, CLI-интерфейс, связывание конвейера.

## Тестирование
* `tests/pass/` — Успешные тесты на нашем языке (проверяют фичи: структуры, массивы, циклы и т.д.).
* `tests/fail/` — Негативные тесты (проверяют отлов семантических и синтаксических ошибок).
* `run_tests.sh` — Главный bash-скрипт для автоматического прогона всех тестов.

## Документация (`docs/`)
* `README.md` — Общее описание проекта и инструкции по запуску.
* `SPECIFICATION.md` — EBNF грамматика и правила системы типов.
* `WALKTHROUGH.md` — Артефакт с пошаговым описанием процесса разработки и принятых решений.
* `FILE_INDEX.md` — Данный файл-карта.

## Инструменты и Архивы
* `tools/` — Вспомогательные утилиты и скрипты, написанные в процессе отладки.
* `archive/patch_backups/` — Свалка для файлов `.orig` и `.rej`, созданных утилитой `patch` при автоматическом разрешении конфликтов кода.
* `Makefile` — Скрипт сборки проекта (команда `make`).
