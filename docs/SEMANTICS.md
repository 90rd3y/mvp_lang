# Семантика и выполнение программы

Этот документ описывает правила поведения, управления памятью, областями видимости и выполнением инструкций интерпретатором языка.

## 1. Архитектура и представление (Data-Oriented AST)

После лексического и синтаксического анализа программа преобразуется в Абстрактное Синтаксическое Дерево (AST). Для максимизации производительности:
* Дерево плоское (Flat AST): узлы хранятся в непрерывном `std::vector<ASTNode>`.
* Связи "родитель-ребенок" реализованы через 32-битные индексы (`NodeId`), а не через указатели (`std::unique_ptr`). Это кардинально повышает Cache Locality (соседние узлы часто находятся в одной линии кэша).
* Семантический анализатор (Semantic Analyzer) делает один проход по AST, разрешая типы, вычисляя смещения в памяти и валидируя корректность семантики. 

## 2. Управление памятью (Arena Allocator)

В рантайме интерпретатора запрещен стандартный `malloc`/`new` (кроме первичной аллокации буферов). 
Вместо этого используется собственный **Arena Allocator (Bump Allocator)**:
1. Выделяется крупный чанк памяти (например, 4 МБ).
2. Выделение памяти в рантайме (создание массива, строки, структуры) — это просто сдвиг указателя (`ptr += size`). Аллокация занимает `O(1)` и выполняется молниеносно.
3. По завершении работы интерпретатора или при уничтожении Арены память очищается целиком без поштучного освобождения объектов (`free`).
4. Это полностью исключает утечки памяти (Memory Leaks) и делает рантайм-выделение крайне эффективным.

## 3. Области видимости и переменные (Scoping)

Язык поддерживает лексические (блочные) области видимости переменных.
* Переменная живет с момента объявления `Тип имя;` до закрывающей скобки блока `}`.
* Вложенные блоки создают новые области видимости (`scopes`).
* Повторное объявление переменной с тем же именем в одной области видимости вызывает семантическую ошибку.
* **Затенение (Shadowing)**: объявление переменной во вложенном блоке перекрывает переменную с тем же именем из родительского блока. После выхода из вложенного блока восстанавливается доступ к родительской переменной.

## 4. Память: Массивы и Сплющивание структур (Flattening)

Семантика аллокации составных объектов оптимизирована для скорости:
* **Массивы**: `целое[5] м;` сразу выделяет непрерывный блок на 5 ячеек типа `Value` в Арене.
* **Структуры**:
  * Интерпретатор не использует дерево объектов с указателями (как в Python/Java).
  * Во время семантического анализа структура "сплющивается" (Flattened Memory Model) до размера массива её примитивных полей.
  * Если `Структура А` содержит `Структура Б` и примитивное поле, размер А равен `размер(Б) + 1`.
  * Доступ к полям (`а.б.в`) в рантайме — это просто прямое обращение по скомпилированному `offset`, равносильное доступу по индексу массива `arr[offset]`. Это делает работу со структурами в языке сопоставимой по скорости с C/C++.

## 5. Управление потоком (Control Flow)

* **Условия (`если/иначе`)**: Условие всегда обязано возвращать строгий логический тип (`лог`). Неявного приведения из целого числа в `лог` (как в C) нет.
* **Циклы (`пока`)**:
  * Циклы выполняются, пока условие истинно.
  * `прервать;` (break) мгновенно прерывает ближайший окружающий цикл.
  * `продолжить;` (continue) переводит выполнение к следующей итерации (перепроверке условия).
* **Функции (`вернуть`)**:
  * Возвращаемое значение функции обязано строго совпадать с заявленным типом возврата.
  * Если функция объявлена как `пусто` (void), разрешен возврат без аргумента `вернуть;` или неявный выход по завершении блока.
  * Аргументы передаются **по значению** (by value). (Массивы и структуры технически передаются как указатели на память Арены, что дает семантику передачи по ссылке для мутаций).

## 6. Встроенные инструкции

Часть функционала зашита в сам язык как встроенные примитивы:
* `печать Expr;` — вычисляет выражение и выводит его в стандартный поток вывода с переводом строки.
* `ввод()` — приостанавливает выполнение и ожидает строку из `stdin`, память под строку выделяется в Арене.
* `паника(String)` — немедленно обрушает интерпретатор с распечаткой переданной ошибки.
* `утверждение(Bool)` — проверяет инвариант; если передан `ложь`, интерпретатор падает (используется для контрактного программирования и тестов).
* `выход(Int)` — штатно завершает интерпретатор с возвратом указанного системного exit-code.
