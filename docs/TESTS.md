# Документация по тестированию MVP Lang

В этом документе описана архитектура тестирования языка, логика запуска тестов и подробное описание каждого теста в системе.

## Логика выполнения тестирования

Для автоматического запуска всех тестов используется Bash-скрипт `run_tests.sh`. Он проверяет корректность работы интерпретатора `./myi`. 

### Порядок работы `run_tests.sh`
1. **Позитивные тесты (Pass):**
   Скрипт перебирает все файлы в директории `tests/pass/*.lang` и запускает их через интерпретатор. Для этих тестов ожидается **успешное выполнение**. Если код возврата процесса равен `0`, тест считается пройденным (`[OK]`). Если код возврата равен `88` (специальный код ошибки AddressSanitizer), тест помечается как `[ASAN FAIL]`. Иначе тест помечается как проваленный (`[FAIL]`).
   
2. **Негативные тесты (Fail):**
   Скрипт перебирает файлы в директории `tests/fail/*.lang`. Для них ожидается **возникновение ошибки** (синтаксической, семантической или рантайм-паники). Если код возврата процесса не равен нулю (`!= 0`) и не равен `88` (утечка памяти), это означает, что интерпретатор корректно отловил ошибку, и тест считается пройденным (`[OK]`). Если интерпретатор завершается с нулем или падает с ошибкой памяти (код `88`), тест провален (`[FAIL]` или `[ASAN FAIL]`).

3. **Проверка аргументов CLI:**
   Скрипт проверяет поведение интерпретатора в некорректных сценариях запуска: без аргументов, с путями к несуществующим файлам, с пустым файлом и с передачей неверных флагов. Во всех случаях ожидается безопасный перехват и информативное сообщение без системных падений.

4. **Сбор статистики:**
   В конце выводится общая статистика — сколько тестов прошло успешно, а сколько упало. Если есть хотя бы один упавший тест, скрипт завершает работу с кодом `1` (что позволяет использовать его в CI/CD).

5. **Поддержка AddressSanitizer (ASan):**
   Для максимальной надежности рантайма и аллокатора памяти (Arena) все тесты (включая C++ модульные тесты и скриптовые `.lang` тесты) можно прогнать со строгим отслеживанием памяти. Запуск производится через `Makefile`:
   `make test-asan`
   Это скомпилирует проект с флагами `-fsanitize=address -fno-omit-frame-pointer -O1` и проверит все тесты на утечки, переполнения буфера и работу с неинициализированной памятью.

## Описание тестов

### Позитивные тесты (`tests/pass/`)
Эти тесты проверяют корректно написанный код. Проверяется, что интерпретатор может без ошибок разобрать и исполнить каждую языковую конструкцию. Ожидаемый результат: код возврата `0`.

| Файл | Назначение и проверяемые механизмы |
| :--- | :--- |
| `01_math.lang` | Базовые математические операции (+), объявление переменных и печать результата. |
| `02_scope.lang` | Работа областей видимости (scope) и затенение (shadowing) переменных во вложенных блоках `{}`. |
| `03_control_flow.lang` | Конструкции управления потоком: цикл `пока` (while) и ветвление `если/иначе` (if/else). |
| `04_functions.lang` | Объявление пользовательских функций, вызов с передачей аргументов и возврат значения. |
| `05_recursion.lang` | Поддержка рекурсивных вызовов на примере функции вычисления факториала. |
| `06_void_func.lang` | Проверка функций без возвращаемого значения (возвращаемый тип `пусто`). |
| `07_loop_control.lang` | Операторы прерывания `прервать` (break) и пропуска итерации `продолжить` (continue) внутри цикла. |
| `08_self_test.lang` | Комплексный тест (логика, рекурсия, вычисления) с использованием системной функции `утверждение` (assert). |
| `09_arrays_and_types.lang` | Поддержка массивов (индексация, чтение/запись элементов) и работа с псевдонимами типов (`тип`). |
| `10_bubble_sort.lang` | Интеграционный алгоритмический тест: реализация пузырьковой сортировки массива во вложенных циклах. |
| `11_structs.lang` | Работа со структурами: объявление, инициализация, обращение к вложенным полям через точку (`.`). |
| `12_full_spec.lang` | Финальный интеграционный тест. Комбинирует всё: типы, вложенные структуры, массивы, циклы и функции. |
| `13_namespaces.lang` | Работа с пространствами имен (`пространство`) и вызов функций по полному пути (например, `Утилиты::запись`). |
| `14_casting.lang` | Явное приведение типов с использованием оператора `как<тип>(...)`. |
| `15_prototypes.lang` | Предварительное объявление функций (прототипы) и их последующая реализация. |
| `16_char_literals.lang` | Символьные литералы (`сим`): инициализация, присваивание и корректный вывод на печать (включая UTF-8 кириллицу). |
| `17_user_types.lang` | Пользовательские псевдонимы типов (алиасы): использование оператора `тип` для сокращения длинных названий типов. |

### Негативные тесты (`tests/fail/`)
Эти тесты содержат намеренные ошибки разного рода. Интерпретатор должен отловить их на этапе анализа или исполнения. Ожидаемый результат: код возврата `!= 0`.

| Файл | Ожидаемая ошибка (что проверяется) |
| :--- | :--- |
| `01_type_mismatch.lang` | Ошибка статической типизации: попытка сложить `целое` и `вещ` без явного приведения типов. |
| `02_undeclared.lang` | Использование переменной, которая не была объявлена. |
| `03_no_main.lang` | Ошибка отсутствия главной функции-точки входа программы (`Начало`). |
| `04_wrong_args_count.lang` | Вызов функции с неверным количеством аргументов (отличным от сигнатуры). |
| `05_wrong_arg_type.lang` | Передача в функцию аргумента неверного типа (`вещ` вместо `целое`). |
| `06_wrong_return_type.lang` | Попытка вернуть из функции значение, тип которого не соответствует заявленному в сигнатуре. |
| `07_uninitialized.lang` | Попытка использовать объявленную, но не инициализированную переменную. |
| `08_recursive_struct.lang` | Объявление рекурсивной структуры (структуры, содержащей саму себя по значению), что приводит к бесконечному размеру памяти. |
| `09_multidim_array.lang` | Попытка использования многомерных массивов `[5][5]`, что запрещено правилами языка. |
| `10_invalid_expr_stmt.lang` | Выражение, не имеющее побочных эффектов, использованное как самостоятельный оператор (например, `1 + 1;`). |
| `11_assign_expr.lang` | Попытка использовать оператор присваивания в качестве выражения (например, `а = б = 5;`). |
| `12_decl_in_block.lang` | Объявление функции внутри вложенного блока кода (например, внутри цикла `пока`). |
| `13_break_outside_loop.lang` | Использование оператора `прервать` (break) вне тела цикла. |
| `14_invalid_comparison.lang` | Применение операторов отношения (например, `>`) к несовместимым типам (к строкам). |
| `15_invalid_bool_comparison.lang`| Применение операторов отношения (например, `<`) к логическим типам (булевым). |
| `16_infinite_recursion.lang` | Тест на превышение максимальной глубины стека вызовов функций (бесконечная рекурсия), приводящее к безопасной панике рантайма. |
| `17_stack_trace.lang` | Проверка генерации стека вызовов (call stack) при возникновении паники внутри глубоко вложенных функций. |
| `18_char_empty.lang` | Ошибка лексера: пустой символьный литерал `''`. |
| `19_char_multichar.lang` | Ошибка лексера: слишком длинный символьный литерал `'abc'`. |
| `20_char_unclosed.lang` | Ошибка лексера: незакрытый символьный литерал. |
| `21_unary_minus_bool.lang` | Статическая ошибка: попытка применить унарный минус (`-`) к значению логического типа. |
| `22_unary_minus_str.lang` | Статическая ошибка: попытка применить унарный минус (`-`) к значению строкового типа. |

### Модульные C++ тесты (`tests/`)
В папке `tests/` располагаются модульные тесты для каждого из компонентов компилятора/интерпретатора. Они написаны на C++ и запускаются автономно. Для негативных тестов (когда ожидается аварийное завершение `exit(1)`) применяется системный вызов `fork()` и анализ кода возврата.

- **`test_memory.cpp`**: Тесты для подсистемы управления памятью (`Memory::Arena`). Проверяют: выделение 0 байт, нестандартное выравнивание, стресс-тесты переключения чанков, сброс арены (`reset`) и корректный вызов паники (panic) при запросе памяти, превышающей размер чанка.
- **`test_lexer.cpp`**: Тесты лексера (токенизатора). Удостоверяются в правильном распознавании типов данных (числа, строки, символы), операторов, ключевых слов, идентификаторов, игнорировании пробелов/комментариев. Дополнительно проверяется генерация токенов ошибки (`Error`) при незавершенных строках и неизвестных символах.
- **`test_parser.cpp`**: Тесты парсера. Проверяют правильность генерации дерева AST, корректный приоритет математических операций (Pratt Parser), вложенность блоков и обработку синтаксических ошибок (ожидание переменной и др.).
- **`test_semantic.cpp`**: Тесты семантического анализатора. Охватывают области видимости (shadowing переменных), проверки типов (совпадение при инициализации, бинарные операции), и контексты выполнения (возврат правильного типа из функции, невозможность изменения констант `пост`). Включают проверку отлова ошибок (необъявленные переменные, неправильные типы).
- **`test_interpreter.cpp`**: Тесты виртуальной машины (VM). Проверяют полное сквозное исполнение кода (AST -> результат): возврат значений из функций, математику, ветвления (`если/иначе`), корректную изоляцию переменных между блоками, а также отлов паник в рантайме (например, деление на ноль).

- **`test.lang`**: Простой демонстрационный файл. Описывает базовые переменные и печать суммы.
