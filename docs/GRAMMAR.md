# Грамматика Языка

Этот документ описывает лексическую структуру и синтаксическую грамматику учебного языка. Для описания используется расширенная форма Бэкуса-Наура (EBNF). 
Внимание: данное описание строго соответствует **текущей реализации** парсера (Parser) и лексера (Lexer).

## 1. Лексическая структура (Lexer)

### 1.1 Токены
Исходный код программы разбивается на последовательность токенов:
- **Ключевые слова**: `целое`, `бцелое`, `вещ`, `лог`, `символ`, `строка`, `пусто`, `если`, `иначе`, `пока`, `прервать`, `продолжить`, `вернуть`, `истина`, `ложь`, `структура`, `тип`, `массив`, `функция`, `пространство`, `как`.
- **Идентификаторы**: имена переменных, функций, структур и типов.
- **Литералы**: числовые (целые и вещественные), строковые и логические константы.
- **Операторы и пунктуация**: `+`, `-`, `*`, `/`, `%`, `=`, `==`, `!=`, `<`, `<=`, `>`, `>=`, `&&`, `||`, `!`, `.`, `,`, `:`, `::`, `;`, `(`, `)`, `{`, `}`, `[`, `]`.

### 1.2 Алфавит и правила наименования
* Идентификаторы состоят из букв (включая кириллицу), цифр и символа подчеркивания `_`.
* Идентификатор не может начинаться с цифры.
* Поддерживается UTF-8 (символы с кодом > 127).
* Язык регистрозависимый (case-sensitive).

```ebnf
Identifier ::= [a-zA-Zа-яА-ЯёЁ_] [a-zA-Zа-яА-ЯёЁ0-9_]*
```

### 1.3 Литералы
```ebnf
IntLiteral    ::= [0-9]+
FloatLiteral  ::= [0-9]+ "." [0-9]+
StringLiteral ::= '"' [^"]* '"'
BoolLiteral   ::= "истина" | "ложь"
```

## 2. Синтаксическая грамматика (Parser)

Грамматика основана на рекурсивном спуске для объявлений и инструкций (statements), и на Pratt-парсере для математических и логических выражений.

### 2.1 Базовые элементы
Типом может выступать как встроенное ключевое слово, так и пользовательский идентификатор.
```ebnf
Type ::= "целое" | "бцелое" | "вещ" | "лог" | "символ" | "строка" | "пусто" | Identifier
```

### 2.2 Структура программы
Программа состоит из набора объявлений на глобальном уровне.
```ebnf
Program ::= Decl* EOF
```

### 2.3 Объявления (Declarations)
```ebnf
Decl ::= NamespaceDecl
       | FuncDecl 
       | StructDecl 
       | TypeAlias 
       | VarDecl 
       | Stmt

NamespaceDecl ::= "пространство" Identifier "{" Decl* "}"

FuncDecl      ::= "функция" Identifier "(" Params? ")" ":" Type (";" | Block)
Params        ::= Param ("," Param)*
Param         ::= Type Identifier

StructDecl    ::= "структура" Identifier "{" StructField* "}"
StructField   ::= Type Identifier ";"

TypeAlias     ::= "тип" Identifier "=" Type ";"

VarDecl       ::= Type ("[" IntLiteral "]")? Identifier "=" Expr ";"
```
*(Примечание: Инициализация переменной `=` является **строго обязательной** в текущей реализации)*

### 2.4 Инструкции (Statements)
Внутри блоков кода (`Block`) допускаются только локальные объявления переменных и инструкции (вызовы, присваивания, циклы и т.д.).
```ebnf
Stmt         ::= IfStmt 
               | WhileStmt 
               | ReturnStmt 
               | BreakStmt 
               | ContinueStmt 
               | Block 
               | AssignStmt
               | ExprStmt

IfStmt       ::= "если" "(" Expr ")" Stmt ("иначе" Stmt)?
WhileStmt    ::= "пока" "(" Expr ")" Stmt
ReturnStmt   ::= "вернуть" Expr? ";"
BreakStmt    ::= "прервать" ";"
ContinueStmt ::= "продолжить" ";"
Block        ::= "{" (VarDecl | Stmt)* "}"
AssignStmt   ::= Expr "=" Expr ";"
ExprStmt     ::= CallExpr ";"
```
*(Примечание 1: Операция присваивания `= ` реализована на уровне инструкций `AssignStmt`, а не в Pratt-парсере)*
*(Примечание 2: Одиночная инструкция-выражение `ExprStmt` допускает **только вызовы функций**, иначе парсер выдает ошибку "Инструкция-выражение может быть только вызовом функции")*

### 2.5 Выражения (Expressions - Pratt Parser)
Выражения парсятся с учетом приоритетов (от самого низкого к самому высокому). 
*(Внимание: согласно текущему коду в `Parser.cpp`, лексер распознает операторы `&&`, `||`, `%`, `!`, но они **не зарегистрированы** в таблице парсинга `get_rule`. Таким образом, грамматика парсера фактически не поддерживает логические операции и остаток от деления).*

```ebnf
Expr         ::= Equality

Equality     ::= Comparison (("==" | "!=") Comparison)*
Comparison   ::= Term (("<" | "<=" | ">" | ">=") Term)*
Term         ::= Factor (("+" | "-") Factor)*
Factor       ::= Unary (("*" | "/") Unary)*

Unary        ::= "-" Unary 
               | Postfix

Postfix      ::= Primary ("(" Arguments? ")" | "[" Expr "]" | "." Identifier)*
CallExpr     ::= Primary ("(" Arguments? ")")+ /* Для ExprStmt */

Primary      ::= IdentifierPath
               | StructLiteral
               | IntLiteral 
               | FloatLiteral 
               | StringLiteral 
               | BoolLiteral 
               | ArrayLiteral
               | CastExpr
               | "(" Expr ")"

IdentifierPath ::= Identifier ("::" Identifier)*
StructLiteral  ::= IdentifierPath "{" (Expr ("," Expr)*)? "}"
ArrayLiteral   ::= "[" (Expr ("," Expr)*)? "]"
CastExpr       ::= "как" "<" Type ">" "(" Expr ")"

Arguments    ::= Expr ("," Expr)*
```

### 2.6 Встроенные функции
Встроенные функции (такие как `печать`, `ввод`, `выход`, `паника`, `утверждение`) не являются ключевыми словами языка на уровне лексера. Они обрабатываются как обычные вызовы функций (`CallExpr`) и разрешаются на этапе семантического анализа и интерпретации как функции из встроенного окружения/библиотеки.
