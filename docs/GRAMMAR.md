# Грамматика языка (EBNF)

Нотация: `::=` определение, `|` альтернатива, `[ ]` опционально, `{ }` повтор (0+), `" "` терминал, `(* *)` комментарий.

---

## 1. Лексическая грамматика

### 1.1 Идентификаторы

```
IDENT      ::= ID_START { ID_CONT }
ID_START   ::= ASCII_LETTER | "_" | NON_ASCII_BYTE
ID_CONT    ::= ASCII_LETTER | ASCII_DIGIT | "_" | NON_ASCII_BYTE
```

`NON_ASCII_BYTE` — любой байт со значением > 127. Проверка UTF-8 структуры не производится, поэтому кириллица (и вообще любые многобайтовые последовательности) принимаются как часть идентификатора «как есть».

### 1.2 Ключевые слова

Зарезервированные слова (не могут быть идентификаторами):

```
целое  вещ  лог  символ  строка  пусто  пост
если  иначе  пока  прервать  продолжить  вернуть
истина  ложь
структура  тип  функция  пространство  как
```

### 1.3 Литералы

```
INT     ::= DIGIT { DIGIT }
FLOAT   ::= DIGIT { DIGIT } "." DIGIT { DIGIT }
STRING  ::= "\"" { ЛюбойСимвол-кроме("\"" | "\\") | ESCAPED_SEQ } "\""
CHAR    ::= "'" ( "\\" ЛюбойСимвол | UTF8_ГРАФЕМА ) "'"
ESCAPED_SEQ ::= "\\" ("\\" | "н" | "п" | "0" | "\'" | "\"")
```

Примечания:
- `FLOAT` формируется, только если сразу после `.` идёт цифра; иначе `.` — отдельный токен `Dot` (т.е. `5.` без цифры после точки не образует float).
- Допустимый набор экранирующих последовательностей в `STRING`/`CHAR`: `\н` (перевод строки), `\п` (табуляция), `\0` (нулевой байт), `\\`, `\'`, `\"`. Любая иная последовательность после `\` — ошибка лексера («Недопустимая экранирующая последовательность»).
- `CHAR` допускает ровно одну «графему»: либо одну (допустимую) экранированную последовательность, либо один ведущий байт + все последующие байты-продолжения UTF-8 (0x80–0xBF) — так поддерживаются кириллические литералы вида `'а'`.

### 1.4 Операторы и разделители

```
+  -  *  /  %
=  ==  !=
<  <=  >  >=
&&  ||  !
.  ,  :  ::  ;
(  )  {  }  [  ]
```

### 1.5 Комментарии (пропускаются, токенов не образуют)

```
LineComment  ::= "//" { ЛюбойСимвол-кроме("\н") }
BlockComment ::= "/*" ... "*/" ;   (* поддерживает вложенность через счётчик глубины *)
```

## 2. Синтаксическая грамматика

```
Program ::= { Declaration } EOF ;

Declaration ::= NamespaceDecl
              | FuncDecl
              | StructDecl
              | TypeAlias
              | VarDecl
              | Statement
```

`Declaration` разрешён и на верхнем уровне программы, и внутри `NamespaceDecl`. Внутри `Block` (тело функции/if/while) допускаются только `VarDecl` и `Statement` — вложенные `функция`/`структура`/`тип`/`пространство` внутри блоков **синтаксически запрещены**.

```
Block ::= "{" { VarDecl | Statement } "}"
```

### 2.1 Пространство имён

```
NamespaceDecl ::= "пространство" IDENT "{" { Declaration } "}"
```

### 2.2 Функция

```
FuncDecl  ::= "функция" IDENT "(" [ ParamList ] ")" ":" TypeSpecLoose ( ";" | Block )
ParamList ::= Param { "," Param }
Param     ::= TypeSpecLoose IDENT
```

`";"` вместо тела — прототип функции (без реализации). Возвращаемый тип обязателен всегда (в т.ч. `пусто` для void-функций).

### 2.3 Структура

```
StructDecl ::= "структура" IDENT "{" Field { Field } "}"
Field      ::= TypeSpecLoose IDENT ";"
```

Требуется минимум одно поле (иначе статическая ошибка «структура должна содержать хотя бы одно поле»).

### 2.4 Синоним типа

```
TypeAlias ::= "тип" IDENT "=" TypeSpecLoose ";"
```

### 2.5 Объявление переменной

```
VarDecl ::= [ "пост" ] BaseType [ "[" INT "]" ] IDENT "=" Expression ";"
```

Инициализация **обязательна** для любой переменной (нет объявления без `= выражение`). Размер массива при объявлении переменной — **обязательное положительное целое число** в скобках.

## 3. Типы

```
BaseType       ::= PrimitiveType | QualifiedName
PrimitiveType  ::= "целое" | "вещ" | "лог" | "символ" | "строка" | "пусто"
QualifiedName  ::= IDENT { "::" IDENT }
```

Два варианта записи размера массива, отличающиеся в разных контекстах:

```
(* параграф 2.5 VarDecl — размер обязателен *)
ArraySuffixStrict ::= "[" INT "]"

(* Параметры функций, тип возврата, поля структур, синонимы типов —
   размер необязателен ("[]" допустимо без числа) *)
TypeSpecLoose ::= BaseType [ "[" [ INT ] "]" ]
```

Многомерные массивы запрещены явной проверкой (`[N][M]` — статическая ошибка), как и вложенные литералы массивов (`[[1,2],[3,4]]`).

### 3.1 Неоднозначность «объявление vs. инструкция» (lookahead парсера)

В начале `Declaration`/операторов внутри `Block` парсер должен решить, является ли идентификатор именем типа (→ `VarDecl`) или частью выражения-инструкции. Правило:

- если текущий токен — примитивное ключевое слово типа (`целое|вещ|лог|символ|строка`, **без** `пусто`) → это тип;
- если текущий токен — `IDENT`, за которым (после произвольной цепочки `:: IDENT`, и опционального `[ [INT] ]`) следует **ещё один** `IDENT` → это тип (имя пользовательского/квалифицированного типа перед именем переменной);
- иначе — это `Statement` (выражение/присваивание/вызов).

Т.е. различение требует просмотра вперёд более чем на 1 токен и не выражается напрямую в контекстно-свободной EBNF выше — это активная эвристика, а не часть формальной грамматики.

## 4. Инструкции (Statement)

```
Statement ::= IfStmt
            | WhileStmt
            | ReturnStmt
            | "прервать" ";"
            | "продолжить" ";"
            | Block
            | ExprStmt

IfStmt     ::= "если" "(" Expression ")" Statement [ "иначе" Statement ]
WhileStmt  ::= "пока" "(" Expression ")" Statement
ReturnStmt ::= "вернуть" [ Expression ] ";"
```

`иначе если (...)` работает как обычная рекурсия (`Statement` после `иначе` сам может быть `IfStmt`), отдельного `elif`-синтаксиса нет.

```
ExprStmt ::= Expression [ "=" Expression ] ";"
```

Семантические ограничения на `ExprStmt` (проверяются сразу в парсере, статическая ошибка при нарушении):

- если `"="` **отсутствует** — корневой узел `Expression` обязан быть вызовом функции (`Call`). Любое иное "голое" выражение-инструкция (`a + b;`, просто `a;`) — статическая ошибка типа "Инструкция-выражение может быть только вызовом функции".
- если `"="` **присутствует** — левая часть обязана быть одним из: `Identifier`, `Indexing` (`a[i]`), `MemberAccess` (`a.b`). Присваивание **не является выражением**: `a = b = c;` не поддерживается (нет права части — второе `=` внутри уже распарсенного выражения `b` не потребляется).

## 5. Выражения

Ниже — эквивалентное представление в виде классической иерархии приоритетов (от низшего к высшему):

```
Expression  ::= LogicOr

LogicOr     ::= LogicAnd { "||" LogicAnd }
LogicAnd    ::= Equality { "&&" Equality }
Equality    ::= Comparison { ( "==" | "!=" ) Comparison }
Comparison  ::= Term { ( "<" | "<=" | ">" | ">=" ) Term }
Term        ::= Factor { ( "+" | "-" ) Factor }
Factor      ::= Unary { ( "*" | "/" | "%" ) Unary }
Unary       ::= ( "!" | "-" ) Unary | Postfix
Postfix     ::= Primary { CallArgs | IndexArgs | MemberArgs }

CallArgs    ::= "(" [ Expression { "," Expression } ] ")"
IndexArgs   ::= "[" Expression "]"
MemberArgs  ::= "." IDENT

Primary ::= INT | FLOAT | STRING | CHAR | "истина" | "ложь"
          | "(" Expression ")"
          | ArrayLiteral
          | CastExpr
          | QualifiedName [ StructLiteralTail ]

StructLiteralTail ::= "{" [ Expression { "," Expression } ] "}"
ArrayLiteral      ::= "[" [ Expression { "," Expression } ] "]"
CastExpr          ::= "как" "<" TypeSpecLoose ">" "(" Expression ")"
```

Примечания:
- `ArrayLiteral` запрещает вложенные литералы массивов в качестве непосредственного элемента (`[[1,2]]` — статическая ошибка), но допускает индексацию/вызовы как элементы (`[a[0], f()]`).
- `CastExpr` находится на уровне `Primary`, поэтому к результату каста применим постфиксный `Postfix`-хвост (`как<целое>(x).метод()`).
- Унарный `+` не существует — только `!` и `-`.
- Составные присваивания и `++`/`--` не поддерживаются на уровне лексера/грамматики.

## 6. Таблица приоритетов (как в коде, `Parser::Precedence`)

| Уровень (снизу вверх) | Операторы            |
|---|---|
| `PREC_OR`         | `\|\|` |
| `PREC_AND`        | `&&` |
| `PREC_EQUALITY`   | `==` `!=` |
| `PREC_COMPARISON` | `<` `<=` `>` `>=` |
| `PREC_TERM`       | `+` `-` (бинарные) |
| `PREC_FACTOR`     | `*` `/` `%` |
| `PREC_UNARY`      | `!` `-` (унарные) |
| `PREC_CALL`       | `(...)` вызов, `[...]` индексация, `.` доступ к полю |

`PREC_ASSIGNMENT` формально существует в enum, но `"="` не зарегистрирован в таблице Pratt-правил ни как prefix, ни как infix — присваивание целиком обрабатывается вне `Expression`, на уровне `ExprStmt` (см. параграф 4).

## 7. Известные ограничения

1. `i++;`, `i--;`, `i += 1;` — не поддерживаются синтаксически (нет соответствующих токенов/правил) → статическая ошибка.
2. `+5` (унарный плюс) — не поддерживается → статическая ошибка «Ожидалось выражение».
3. Побитовые `&`, `|` — отдельные символы дают ошибку лексера, поддерживаются только `&&`/`||`.
4. Отдельного ключевого слова для массива в языке нет — массивы объявляются исключительно суффиксом `[N]` у типа (см. параграф 3).
5. Голые выражения-инструкции без вызова функции (`a;`, `a + b;`) запрещены — инструкция-выражение обязана быть вызовом функции.
6. Присваивание не является выражением: нельзя писать `a = b = c;`, `если (a = b)` и т.п.
7. Размер массива обязателен при объявлении переменной (`целое[5] arr = ...;`), но необязателен в параметрах функций/типе возврата/полях структур/синонимах типов (`целое[] arr` допустимо там).
8. Многомерные массивы и вложенные литералы массивов запрещены явными проверками парсера.
9. Экранирование в строковых/символьных литералах ограничено набором `\н \п \0 \\ \' \"`; любая иная последовательность после `\` — ошибка лексера.
10. Внутри `Block` (тело функции, if/while) нельзя объявлять `функция`/`структура`/`тип`/`пространство` — только переменные и инструкции.
