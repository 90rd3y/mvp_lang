# ТЗ семестрового проекта по разработке языка программирования и его интерпретатора

## Описание проекта

Целью проекта является разработка собственного языка программирования и реализация интерпретатора для него на языке C++.

Ключевые характеристики языка:
* сильная, статическая типизация
* русский синтаксис
* минималистичен

Необходимо самостоятельно проектировать синтаксис и семантику языка, фиксировать их в спецификации, реализовывать pipeline интерпретации, включающий лексический анализ, синтаксический анализ, семантическую проверку и прямое исполнение AST в собственной среде выполнения.

Ожидается работающая учебная реализация. В рамках жесткого контроля времени и ресурсов проект строго ограничен минимально необходимым функционалом.

## Требования к языку программирования

### Лексика

Определяются алфавит языка и лексические единицы в формате EBNF.

Язык обязан поддерживать:
- **Ключевые слова** — зарезервированы и не могут использоваться как идентификаторы.
- **Идентификаторы** — имена переменных, функций, типов, пространств имён.
- **Литералы** — целые и вещественные числа, символы, строки, булевы значения.
- **Escape-последовательности** — строковые и символьные литералы поддерживают экранирование; минимальный набор - аналоги следующих экранируемых символов: `\n`, `\t`, `\\`, `\"` (и `\'` для символьных литералов); конкретный набор фиксируется в спецификации.
- **Операторы и разделители**.
- **Комментарии** — однострочные и блочные многострочные комментарии (не влияют на семантику программы). Поддержка многострочных комментариев, заключенных между двумя специальными последовательностями символом, например, начинающихся с `/*` и заканчивающихся `*/`, с корректной обработкой вложенности комментариев (например, `/* внешний /* вложенный */ комментарий */`). Лексер корректно отслеживает позиции строк внутри комментариев. Конкретные специальные последовательности символов для написания комментариев фиксируются в спецификации языка.


### Синтаксис

Определяется полная грамматика языка в форме EBNF.

Грамматика обязана покрывать:
- структуру программы и точку входа (типа `main`);
- объявления верхнего уровня — переменных, функций, базовых структур, синонимов типов, пространств имён;
- выражения — арифметические, логические, сравнения, индексирование статических массивов и строк, доступ к полям структур, разрешение имён, приведение типов;
- конструкции управления потоком — блоки, `if/else`, `while`, `break`, `continue`, `return`, нулевая инструкция (инструкция без эффекта);
- присваивание — как инструкция или как выражение (фиксируется в спецификации);
- литералы статических массивов и структур.

Выражения и инструкции допускаются только внутри тела функций; на верхнем уровне допускаются только объявления.

### Семантика

Определяется семантика всех конструкций языка.

#### 1. Статическая семантика (Этап анализа)

Определяет правила, которые проверяются до выполнения программы.

* **Область видимости и время жизни:** Переменная существует в пределах блока, в котором она объявлена.
* **Разрешение имён:** Используется лексическая область видимости (lexical scoping). Идентификатор разрешается в ближайшей охватывающей области видимости. Перекрытие имён (shadowing) во вложенных блоках допускается. Одно и то же имя не может быть объявлено дважды в одной области видимости (перегрузки отсутствуют).
* **Статические ошибки:** При выявлении нарушения любого из правил статической семантики (попытка обращения к необъявленной переменной, повторное объявление имени в одном блоке) семантический анализатор фиксирует статическую ошибку.

#### 2. Динамическая семантика (Этап выполнения)

Определяет поведение программы в рантайме.

* **Стратегия вычисления:** Фиксируется как call-by-value (аргументы передаются в функцию по значению) или call-by-reference (по ссылке) — согласно итоговой спецификации.
* **Семантика присваивания:** Value semantics (копирование по значению) или reference semantics — согласно итоговой спецификации.
* **Управление памятью:** Память для данных, время жизни которых превышает один блок, выделяется простейшим собственным аллокатором (например, bump-аллокатором).
* **Побочные эффекты:** Функциям разрешено иметь побочные эффекты (ввод/вывод, изменение мутабельных переменных).
* **Ошибки времени выполнения (Runtime Errors):** При выявлении нарушения логики в процессе выполнения (деление на ноль, выход за границы массива и др.) возникает ошибка времени выполнения. Программа немедленно аварийно завершается с ненулевым кодом возврата (формат логов и трассировки определяется требованиями к интерпретатору).

### Типы и система типов

Язык обязан поддерживать строго следующий фиксированный набор встроенных типов:

| Тип | Размер | Описание |
|-----|--------|----------|
| `int` | строго 64 бит | знаковое целое число |
| `uint` | строго 64 бит | беззнаковое целое число |
| `float` | строго 64 бит (double) | вещественное число двойной точности (IEEE 754) |
| `bool` | — | булево значение (`true` / `false`) |
| `char` | 8 или 32 бит | символ (ASCII или Unicode — фиксируется в спецификации) |
| `string` | — | строка (последовательность символов) |
| `void` | — | отсутствие значения / пустое значение (для возвращаемого типа функций) |
| `array` | — | статический одномерный массив фиксированного размера (размер является частью типа) |
| `struct` | — | базовая пользовательская структура данных (только поля, без методов) |

Конкретные названия типов устанавливаются спецификацией.

Система типов обязана определить следующие правила:
- **Номинальность** или **структурность** — два типа совместимы только если их имена совпадают (номинальность) или совпадает их внутренняя структура (структурность).
- **Строгость типов** — уровень строгости определяется спецификацией языка.
- **Мутабельность** — механизм указания мутабельности переменных (например, `let` для неизменяемых и `var` для изменяемых) определяется спецификацией языка.

**Целые типы (`int`, `uint`):**
- Поддерживаются строго только 64-битные знаковые (`int`) и беззнаковые (`uint`) целые числа.
- Поведение при переполнении определяется спецификацией языка (wraparound, saturating или ошибка времени выполнения).
- Поддерживаются литералы целых чисел в десятичной, шестнадцатеричной и двоичной нотации.

**Вещественные типы (`float`):**
- Поддерживается строго один 64-битный тип для представления вещественных чисел по стандарту IEEE 754 double precision.
- Поддерживаются литералы вещественных чисел, в том числе с экспоненциальной нотацией.
- Поддерживается представление бесконечности (`inf`, `-inf`) и нечислового значения (`NaN`).

**Булевы типы (`bool`):**
- Поддерживается тип для представления логических значений (`true` / `false`).
- Поддерживаются логические операции: конъюнкция, дизъюнкция, отрицание.
- Использование булевых типов в арифметических операциях и числовых типов в логических операциях запрещено.

**Тип `void`:**
- Используется для обозначения отсутствия значения.
- Должны поддерживаться функции с типом возврата `void` (и инструкция `return;` без значения).

**Символы (`char`) и строки (`string`):**
- Символы поддерживают сравнение на равенство и неравенство. Символы не являются арифметическими типами.
- Строки представляют собой последовательность символов. Поддерживаются: конкатенация, получение длины, сравнение на равенство/неравенство по содержимому, индексирование (доступ к символу по позиции).

**Статические массивы (`array`):**
- Поддерживаются строго только статические одномерные массивы фиксированного размера, где размер задается константой на этапе интерпретации и является частью типа (например, `int[10]`).
- Операции: индексирование, получение длины массива, сравнение на равенство/неравенство.

**Базовые структуры (`struct`):**
- Поддерживаются пользовательские составные типы, содержащие исключительно именованные поля заданных типов.
- Операции: доступ к полям по имени, инициализация, сравнение.

**Синонимы типов (`type`):**
- Поддерживается объявление альтернативного имени для существующего типа (например, `type MyInt = int;`).
- Циклы синонимов и ссылки на необъявленные типы диагностируются как статическая ошибка.

### Выражения

Язык поддерживает строго следующий набор выражений:
- **Унарные:** унарный минус, логическое отрицание.
- **Арифметические:** сложение, вычитание, умножение, деление, остаток от деления.
- **Логические:** конъюнкция, дизъюнкция, отрицание.
- **Сравнения:** равенство, неравенство, меньше, больше, меньше или равно, больше или равно.
- **Приведение типов (cast):** явное приведение одного типа к другому.
- **Доступ:** индексирование статического массива, доступ к полю структуры, доступ к элементу в пространстве имён.
- **Литералы составных типов:** литерал массива, литерал структуры, литералы чисел, строк, символов и булевых значений.

Приоритет и ассоциативность операторов фиксируются в спецификации.

### Управление потоком

Язык поддерживает базовые конструкции управления потоком:
- **Блоки** — последовательность инструкций в фигурных скобках, вводящая новую область видимости.
- **Нулевая инструкция** — пустая инструкция `;`.
- **Ветвление** — условная конструкция `if` / `else` (ветка `else` опциональна для пользователя).
- **Циклы** — цикл с предусловием `while`, инструкции `break` (завершение цикла) и `continue` (переход к следующей итерации).
- **Возврат из функции** — инструкция `return <expr>;` (или `return;` для `void`-функций).

### Объявления

Язык поддерживает следующие объявления:
- **Переменные** — объявление именованного значения с указанием типа, начального значения и мутабельности.
- **Функции** — объявление именованной функции с параметрами, типом возврата и телом. 
- **Точка входа** — программа обязана содержать функцию `main` без параметров с типом возврата `int`. Возвращаемое значение передается как код завершения процесса.
- **Структуры** — объявление имени структуры и списка её полей.
- **Синонимы типов** — объявление синонима типа.
- **Пространства имён** — объявление области видимости для группировки объявлений.

### Встроенные функции

- **Ввод/вывод:** `print` (вывод значения в стандартный поток), `input` (чтение строки из стандартного потока).
- **Управление выполнением:** `exit` (завершение программы с кодом возврата), `panic` (аварийное завершение с сообщением), `assert` (требование выполнения условия для продолжения выполнения программы).

## Требования к интерпретатору

### Интерфейс командной строки

Интерпретатор запускается из командной строки:
```
myi <source_files>
```
Конкретное название команды определяется во время или после реализации.

Поддерживаемые диагностические флаги:
- `--dump-tokens` — вывести поток токенов и завершить работу.
- `--dump-ast` — вывести структуру AST и завершить работу.

Код возврата:
- `0` — успешное выполнение программы.
- Ненулевой код — статическая ошибка или ошибка времени выполнения.

### Диагностика ошибок

Все диагностические сообщения выводятся в `stderr` в формате:
```
<файл>:<строка>:<столбец>: ошибка: <сообщение>
```
При возникновении любой ошибки времени выполнения (деление на ноль, выход за границы статического массива, вызов `panic`, сбой `assert`) интерпретатор выводит полную цепочку активных вызовов функций в `stderr` от точки входа `main` до места сбоя с указанием имен функций и номеров строк.
